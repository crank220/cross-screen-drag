# 跨页面拖拽功能文档与注意事项

## 1. 功能目标

本项目根据共享链接中的需求实现一个不依赖 Electron 的 Vue3 + Node 跨页面拖拽示例：

- 左侧为资源列表页 `/resource`。
- 右侧为场景编辑页 `/canvas`。
- 两个页面通过父级工作台 iframe 组合，模拟 DOM 隔离的跨页面场景。
- 支持长按或移动阈值触发拖拽。
- 支持从资源页拖到场景页，也支持从场景页拖回资源页。
- 拖拽素材来自 `assets/`，包括图片、视频、PDF、Excel 数据源，并扩展了基于 PDF/图片素材派生的文本与网页类型。

## 2. 技术选型

| 模块 | 技术 | 说明 |
| --- | --- | --- |
| 前端 | Vue3 + TypeScript + Vite | 负责工作台、资源页、画布页和交互状态 |
| 服务端 | Node.js + Express | 提供素材清单、静态资源、本地网页素材页面 |
| 实时同步 | WebSocket (`ws`) | 同步画布图层新增、移动、删除和回流事件 |
| 跨页面通信 | `postMessage` | 父工作台与两个 iframe 页面交换拖拽事件 |
| 拖拽实现 | Pointer / Mouse Events | 不依赖 HTML5 `dragstart/drop` |

## 3. 页面结构

| 路径 | 组件 | 功能 |
| --- | --- | --- |
| `/` | `Workbench.vue` | 父级工作台，内嵌资源页和场景页 iframe，负责全局拖拽预览与落点判断 |
| `/resource` | `ResourcePage.vue` | 展示可拖拽素材，接收从画布回流的素材 |
| `/canvas` | `CanvasPage.vue` | 展示场景画布，接收资源素材并创建图层 |

## 4. 拖拽流程

1. 资源卡片或画布图层触发 `pointerdown`。
2. `createFrameDragController` 在长按或移动超过阈值后向父页面发送 `drag:start`。
3. 父页面显示全局拖拽预览，并根据 iframe 边界判断当前 hover 目标。
4. 拖拽激活后，父页面临时关闭 iframe 的 `pointer-events`，让移动和释放事件回到父层。
5. 释放时父页面向目标 iframe 发送 `drag:drop`。
6. 目标页处理 drop：
   - 资源页 -> 场景页：`CanvasPage` 通过 WebSocket 发送 `scene:add`。
   - 场景页 -> 资源页：`ResourcePage` 通过 WebSocket 发送 `scene:remove` 与 `library:return`。
7. Node WebSocket 服务广播最新场景状态。

## 5. 素材映射

| 类型 | 素材来源 | 页面呈现 |
| --- | --- | --- |
| `image` | `assets/poster.png` | 图片图层 |
| `video` | `assets/0d44e1e4-289e-4934-a8e5-bacd9399f197-T720P.mp4` | 视频图层 |

| `datasource` | `assets/1.xls` | 表格数据卡片 |

## 6. 服务接口

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| `/api/materials` | `GET` | 返回素材清单 |
| `/api/scene` | `GET` | 返回当前内存中的画布图层 |
| `/api/scene/reset` | `POST` | 清空画布并广播快照 |
| `/assets/*` | `GET` | 静态素材访问 |
| `/asset-site/poster` | `GET` | 基于 `poster.png` 的本地网页素材 |
| `/ws` | WebSocket | 场景和回流事件同步 |

## 7. 运行方式

```bash
npm run dev
```

默认地址：

```text
http://127.0.0.1:5173
```

使用 `127.0.0.1` 是为了避免本机已有项目占用 `localhost:5173` 的 IPv6 回环地址。

内网访问：

```bash
npm run dev:lan
```

该命令会监听 `0.0.0.0`，并在终端输出可供同一局域网设备访问的地址，例如：

```text
LAN access:
  http://192.168.x.x:5173
```

如果当前已有 `npm run dev` 占用 `5173`，需要先停止旧服务，再运行 `npm run dev:lan`。也可以临时换端口：

```bash
PORT=5174 npm run dev:lan
```

## 8. 验证结果

已完成以下验证：

- `npm run build` 通过。
- `npm run dev` 可启动 Node + Vite 服务。
- `npm run dev:lan` 可监听内网地址；本次用 `PORT=5174 npm run dev:lan` 验证通过。
- 资源页拖拽 `poster.png` 到场景页后，`/api/scene` 出现 `poster-image` 图层。
- 场景页图层拖回资源页后，`/api/scene` 清空，资源页显示“从场景回流”提示。
- 已用 Browser 与 Computer Use 对 UI 做过可视化验证。

## 9. 注意事项

1. 当前实现的“跨页面”指同一父窗口下的两个同源 iframe 页面，不是两个完全独立的浏览器窗口。
2. 不使用 Electron 时，浏览器无法像桌面应用那样稳定拿到跨独立窗口的全局鼠标坐标；如果要做跨浏览器窗口或跨物理屏幕拖拽，需要额外设计确认式投放、原生 DnD 兜底或浏览器扩展能力。
3. `postMessage` 只能传递可结构化克隆的数据。本项目在 iframe 通信前会把拖拽 payload 转成纯 JSON，避免 Vue reactive proxy 导致 `DataCloneError`。
4. 拖拽激活后需要临时关闭 iframe 的 `pointer-events`，否则鼠标释放可能落在目标 iframe 内部而不回到父页面。
5. 当前 Node 场景数据保存在内存中，服务重启后会丢失；生产环境需要持久化到数据库或业务后端。
6. `/assets` 只服务本项目素材目录，不接受用户上传路径；新增素材时应同步更新服务端 `materials` 清单。
7. `website` 类型目前是本地演示页面，生产环境接入外部站点时需要处理 X-Frame-Options、CSP、跨域与安全白名单。
8. 内网访问要求访问设备与开发机处于同一 Wi-Fi 或同一局域网；如果无法打开终端输出的地址，需要检查系统防火墙是否允许 Node 监听端口。
